# pia

Pi coding agent profile launcher. Run multiple Pi configurations (different
models, extensions, skills) without manually swapping config files.

## Requirements

- [Pi coding agent](https://github.com/mariozechner/pi-coding-agent) installed and in PATH
- At least one profile definition (see Configuration)

## Usage

```bash
pia profile <name> [pi-args...]   # Launch pi with a named profile
pia sync                          # Generate/update profile directories
pia list                          # List available profiles
pia show <profile>                # Show profile configuration
pia update                        # Update pi + packages for all profiles
pia doctor                        # Check profile health
```

## Configuration

Profiles live at `~/.pi/agent/profiles/<name>/profile.json`.

### Profile Schema

```json
{
  "model": "model-identifier",
  "thinking": "medium",
  "extensions": [
    "~/.pi/agent/extensions/my-extension"
  ],
  "packages": [
    "npm-package-name"
  ],
  "skills": [
    "~/.agents/skills/my-skill"
  ],
  "settings": {
    "any": "additional pi settings"
  },
  "sessionDir": "~/.pi/sessions/profile-name"
}
```

### Fields

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `model` | string | yes | Model identifier for Pi |
| `thinking` | string | no | Thinking mode: `none`, `low`, `medium`, `high` |
| `extensions` | string[] | no | Paths to Pi extension directories |
| `packages` | string[] | no | Pi npm packages to install |
| `skills` | string[] | no | Paths to skill directories |
| `settings` | object | no | Merged into the generated settings.json |
| `sessionDir` | string | no | Session history location |

Tilde (`~`) in paths is expanded to `$HOME` automatically.

### Example: minimal profile

```json
{
  "model": "claude-sonnet-4-20250514",
  "thinking": "medium",
  "packages": ["web-access"]
}
```

### Example: full profile with extensions and skills

```json
{
  "model": "claude-opus-4-20250514",
  "thinking": "high",
  "extensions": [
    "~/.pi/agent/extensions/memory",
    "~/.pi/agent/extensions/status-bar"
  ],
  "packages": ["web-access", "subagents", "intercom"],
  "skills": [
    "~/.agents/skills/go-dev",
    "~/.agents/skills/tdd"
  ],
  "sessionDir": "~/.pi/sessions/full"
}
```

## How Sync Works

`pia sync` generates a working Pi agent directory for each profile at `~/.pi/agent-<name>/`:

1. Creates the directory if it doesn't exist
2. Writes `settings.json` from the profile definition
3. Recreates extension symlinks pointing to the source paths

**Non-destructive**: `sessions/` and `npm/` directories are preserved across
syncs. Only `settings.json` and extension symlinks are overwritten.

## Generated Structure

```
~/.pi/
├── agent/                    # default pi config (not managed by pia)
├── agent-quick/              # generated by pia sync
│   ├── settings.json         # ← written from profile.json
│   ├── extensions/           # ← symlinks recreated
│   ├── npm/                  # ← preserved
│   └── sessions/             # ← preserved
└── agent-research/           # another profile
```

## Shell Aliases

```bash
alias piq='pia profile quick'
alias pir='pia profile research'
```
